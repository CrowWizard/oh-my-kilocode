# 工作流目标

用于处理错误、异常行为、测试失败或线上问题的最小闭环调试流程。

## 模式要求

- 问题归档、根因假设与验证定位阶段可在只读模式进行
- 一旦进入最小修复、回归保护或附加观测补齐阶段，应切换到可写执行模式
- 复盘输出如依赖修复结果、测试结果与变更归档，应保持可写执行模式直到 workflow 收口完成

## 工具偏好

- 涉及代码路径定位、引用分析、根因影响范围判断与精确修复时，优先使用 `serena`

## 角色分层

### 主角色

- `debugger`：承担问题定位、根因假设与修复路径主线

### 辅助角色

- `code-reviewer`：负责检查修复是否引入额外副作用或复杂度问题
- `test-engineer`：负责回归保护与最小验证补齐

## Workflow

### 步骤 1：问题归档
- 收集错误信息、日志、堆栈、复现步骤和环境信息
- 明确影响范围与紧急程度

### 步骤 2：根因假设
- 调用 `debugger` 生成并排序根因假设
- 为每个假设定义证据和证伪方式

### 步骤 3：验证与定位
- 通过代码阅读、日志、断点、附加观测或最小复现验证假设
- 锁定最可能根因

### 步骤 4：最小修复
- 以最小改动修复根因
- 必要时请 `code-reviewer` 检查修复是否引入副作用

### 步骤 5：回归保护
- 由 `test-engineer` 增加最小回归测试或验证步骤
- 确保问题不会重复出现

### 步骤 6：复盘输出
- 输出根因、证据、修复方式、验证结果与预防建议
