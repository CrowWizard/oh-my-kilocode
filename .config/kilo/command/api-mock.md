# 工作流目标

用于在真实服务尚未就绪、第三方接口不可控或测试环境成本过高时，快速建立可信的 API 模拟层。

## 模式要求

- 契约范围收敛、匹配规则设计与场景定义阶段可在只读模式进行
- 一旦进入 mock 文件生成、场景数据写入、测试入口补齐或脚本配置调整阶段，应切换到可写执行模式
- 契约一致性验证与 Mock 说明归档如依赖本地执行结果或文件更新，应保持可写执行模式直到 workflow 收口完成

## Workflow

### 步骤 1：收敛契约范围
- 确认要模拟的接口清单、方法、路径、鉴权方式与核心字段
- 标记必需场景：成功、空结果、校验失败、权限失败、限流、上游异常

### 步骤 2：定义匹配规则
- 为路径、query、header、body 与状态前置条件定义匹配策略
- 明确默认响应、优先级与未命中时的兜底行为

### 步骤 3：设计响应与状态
- 为每个接口定义响应体、状态码、错误结构与关键响应头
- 如需模拟业务流，加入最小状态管理与场景切换能力

### 步骤 4：注入真实感约束
- 根据需要加入延迟、分页、幂等冲突、重试抖动或偶发失败
- 保持模拟行为可控、可重放，避免随机结果影响调试

### 步骤 5：验证契约一致性
- 用示例请求验证 mock 是否符合现有文档、前端调用或测试预期
- 记录哪些字段是占位数据，哪些行为尚未覆盖

### 步骤 6：交付 Mock 说明
- 输出 mock 入口、场景切换方式、已覆盖接口、限制项与后续替换边界
