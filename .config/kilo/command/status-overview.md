# 工作流目标

用于在多阶段任务推进过程中，快速汇总当前进度与阻塞，避免上下文漂移和阶段失焦。

## 模式要求

- 本 workflow 默认可在只读模式完成
- 只有当用户明确要求同步修正文档、计划或状态文件时，才切换到可写执行模式

## 工具偏好

- 涉及长任务阶段、稳定边界或需跨轮次延续的状态时，优先读取或更新 `memory`

## Workflow

### 步骤 1：确认当前目标
- 明确当前正在推进的任务主题
- 判断任务属于 Discovery、Planning 还是 Execution 阶段
- 如当前正在做 `decision-framing`，应归类为 Discovery 向 Planning 过渡时的增强活动，而不是新的 phase

### 步骤 2：汇总已完成项
- 列出已完成的分析、计划、实现、验证或文档更新
- 标注哪些结果已被确认，哪些仍是暂定结论

### 步骤 3：识别阻塞与风险
- 汇总当前阻塞点、信息缺口、失败项与外部依赖
- 标出最值得优先处理的风险

### 步骤 4：给出下一步建议
- 明确下一步应继续哪个 workflow 或 agent
- 如需切换到实现阶段，提醒进入可写执行模式

## 产出清单

- 当前任务目标
- 当前阶段判断
- 已完成项
- 阻塞点与风险
- 下一步建议
